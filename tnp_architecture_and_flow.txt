# TalentBridge TnP Ecosystem: Architectural Specification & Data Flow

## 1. Overview
The Training & Placement Officer (TPO) Ecosystem is a multi-tenant, AI-powered intelligence platform built to bridge the gap between institutional talent and market requirements. It provides TPOs with granular monitoring, automated assessments, and AI-driven skill gap analysis.

## 2. Core Architecture
- **Tenant Isolation**: Data is partitioned by `college_id`. TPOs are assigned to one or more colleges via the `tpo_colleges` mapping table.
- **Unified Query Layer**: A single DB interface in `server/db.ts` handles both MySQL (Production) and SQLite (Local Fallback) using parameterized queries to prevent SQL Injection.
- **RBAC Security**: 
  - Roles: `STUDENT`, `COMPANY`, `TPO`, `ADMIN`, `SUPER_ADMIN`.
  - Middleware: `authenticate` (Identity) + `authorize(['TPO'])` (Role Permission).
  - Row-Level Security: Every TPO controller enforces a `WHERE college_id IN (assigned_ids)` constraint.

## 3. Schema Design (TnP Specific)
- `college_master`: Primary institutional registry.
- `tpo_profiles`: Extended metadata for placement officers (First-login flag, status).
- `tpo_colleges`: Many-to-many link between TPOs and Colleges.
- `tpo_tests`: Custom assessment definitions (Questions stored as JSON for flexibility).
- `student_test_submissions`: Tracking of performance on TPO-created tests.
- `college_analytics`: Cached aggregates of placement rates and skill scores for high-speed dashboard loading.

## 4. Real-Time Data Pipelines
### A. Dashboard Synchronization
1. **Trigger**: Student completes a mock interview or coding quiz.
2. **Backend**: `talent_scores` are updated.
3. **Frontend**: TPO Dashboard fetches `dashboard-stats` which uses SQL aggregates (AVG/SUM) across all students in assigned colleges.
4. **Result**: TPO sees immediate changes in "Placement Readiness" and "At-Risk" metrics.

### B. AI Skill Gap Analysis
1. **Aggregation**: Backend fetches `skills_json` from all students in the cohort.
2. **Market Matching**: Fetches `ACTIVE` job descriptions from the `jobs` table.
3. **Intelligence**: Feeds both datasets into Gemini Pro with a structured prompt.
4. **Action**: Returns JSON containing Missing Skills (e.g., "72% require React") and a 4-phase Training Roadmap.

## 5. Security & Safeguards
- **JWT Boundaries**: Access tokens expire in 24h (Development) or 1h (Production). Refresh tokens handle session continuity.
- **Password Enforcement**: First-time TPOs are intercepted by `ForcePasswordChange.tsx` requiring:
  - 8+ Characters
  - Uppercase + Lowercase
  - Number + Special Character
- **Data Isolation**: The `getTPOContext` helper is the mandatory entry point for all TPO routes, preventing horizontal privilege escalation.

## 6. Real Student Actions -> TPO Stats Flow
[Action: Student Completes Profile] -> [DB: completeness_score updated] -> [Dashboard: "Active Students" count increases]
[Action: Student Selected in Drive] -> [DB: event_registrations status='SELECTED'] -> [Dashboard: "Placed Students" & "Placement Rate" update]
[Action: Student Fails Assessment] -> [DB: talent_score < 40] -> [Dashboard: "At-Risk Students" count increases]
